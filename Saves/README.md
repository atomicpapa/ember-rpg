# Saves

One file per in-progress scenario. The AI writes here **automatically** at every natural checkpoint so a scenario can span many sessions without losing the thread — you never have to remember to save.

A save file is a structured snapshot: current scene, story-so-far recap, live Harm/Conditions/Clocks, NPCs on stage, open threads, and any character changes not yet written to the main character file. To resume, paste the Core rules, the Genre Pack, your Character file, and the relevant Save file. See the persistence rules in `EMBER — Core Engine.md`.
